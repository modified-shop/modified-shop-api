# Security Policy

## Reporting a vulnerability

Do not report security vulnerabilities in a public issue.

Send the report to `info@modified-shop.org`. You can also use GitHub private vulnerability reporting if it is enabled for this repository.

Include:

- The affected REST API version
- Clear reproduction steps
- The possible impact
- A proposed fix, if available

Do not send real shop credentials, API tokens, customer data, certificates, or provisioning profiles.

## Supported versions

Security fixes target the current development version on the `main` branch. Historical snapshots remain available for reference but do not receive security updates.
